# 宝塔云服务器部署与排障说明

## 1. 部署前准备

服务器建议环境：

- `Ubuntu 22.04`
- `Node.js 20+`
- `MySQL 8`
- 已安装 `PM2` 或宝塔 Node 项目管理器

必须准备的环境变量至少包括：

- `NODE_ENV=production`
- `PORT=5600`
- `APP_URL`
- `APP_DOMAIN`
- `APP_STORAGE_MODE=mysql`
- `DATABASE_URL`
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `API_FOOTBALL_KEY`
- `THE_ODDS_API_KEY`

## 2. 首次部署步骤

在项目目录执行：

```bash
npm install
npx prisma generate
npx prisma migrate deploy
npm run build
```

如果数据库还没有导入旧数据，再执行：

```bash
npm run db:import:mysql
npm run db:verify:mysql
```

然后启动服务：

```bash
npm run start
```

如果使用 `pm2`：

```bash
pm2 start dist/server.cjs --name worldcup-app
pm2 save
```

## 3. 生产巡检

### 3.1 数据库巡检

```bash
npm run ops:diagnose
```

重点看这些字段：

- `counts.teams`
- `counts.matches`
- `counts.players`
- `counts.teamHistory`
- `matches.orphanTeamRefs`
- `recentSyncLogs`

### 3.2 健康检查

浏览器或命令行访问：

```bash
curl http://127.0.0.1:5600/api/health
```

### 3.3 管理后台状态检查

先登录后台，再请求：

```bash
GET /api/admin/system/status
GET /api/admin/integrations/status
```

其中 `system/status` 会返回：

- 当前存储模式
- 关键表记录数
- 比赛状态统计
- 缺失球队关联数
- 最近同步日志

## 4. 手动触发赛程同步

### 4.1 同步日期窗口

推荐使用新接口：

```bash
POST /api/admin/sync/window
```

可选参数：

```json
{
  "anchorDate": "2026-06-12",
  "pastDays": 1,
  "futureDays": 10
}
```

### 4.2 只同步单日

```bash
POST /api/admin/sync/fixtures
```

带日期时只同步该天：

```json
{
  "date": "2026-06-12"
}
```

不带日期时，会自动同步默认窗口。

### 4.3 同步赔率

```bash
POST /api/admin/sync/today
```

这个接口会执行：

- 赛程窗口同步
- 赔率同步

## 5. 常见问题排查

### 5.1 球队资料不显示

先执行：

```bash
npm run ops:diagnose
```

如果以下任一项偏小或为 `0`，优先处理数据：

- `counts.teams`
- `counts.players`
- `counts.teamHistory`

然后再检查：

- `GET /api/teams/:id/detail` 是否返回 `team`
- `staticProfile` 是否为 `null`
- `matches.orphanTeamRefs` 是否大于 `0`

### 5.2 赛程、VS、日期、比分不显示

重点检查：

- `counts.matches` 是否正常
- `matches.byStatus` 是否有值
- `fixtureSyncedMatches` 是否持续为 `0`
- 最近 `recentSyncLogs` 是否一直失败

如果赛程缺失，先执行：

```bash
POST /api/admin/sync/window
```

### 5.3 管理后台进不去

先确认：

- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- 浏览器本地缓存是否残留旧 token

当前版本已经统一为“进入后台页后登录”，如果仍异常，清掉浏览器本地存储里的：

- `wc_admin_token`

然后重新进入后台页登录。

### 5.4 MySQL 有数据但页面还是空

优先排查是否导入不完整：

```bash
npm run db:verify:mysql
npm run ops:diagnose
```

如果 `teams / matches / players / team_history` 数据明显不全，重新导入旧数据后再重启服务。

## 6. 更新发布流程

每次更新代码后建议执行：

```bash
npm install
npx prisma generate
npx prisma migrate deploy
npm run build
pm2 restart worldcup-app
```

更新后立刻执行一次：

```bash
npm run ops:diagnose
```

如果诊断结果正常，再访问前台和后台做人工验收。
