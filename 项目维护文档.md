## 1. 项目概述

**2026 美加墨世界杯朋友群观赛竞猜平台** — 面向微信群/朋友群的全栈竞猜 Web 应用。

### 核心功能

| 功能 | 说明 |
|------|------|
| 赛事竞猜 | 胜平负 (H2H)、正确比分 (Correct Score)、总进球 (Total Goals)、晋级、冠军、金靴、金球 |
| AI 分析 | 多提供商路由 (DeepSeek/Mimo/Gemini)，赛前预测、赛后复盘、智能推荐、分享卡生成 |
| 实时推送 | WebSocket 比分更新、竞猜结果、徽章解锁、连胜达成、赔率变动 |
| 排行榜 | 总积分/今日/胜率/连胜/盈利 五榜，AI 自动生成点评文案 |
| 社交互动 | 竞猜道具卡 (4种)、称号与徽章系统、群内动态时间线 |
| 管理后台 | 用户管理、积分调账、头像上传、数据备份恢复、手动赔率同步 |
| 历史长廊 | 22 届世界杯完整数据、48 队 1246 人球员资料 |

### 使用场景

- **手机浏览器**直接访问（非 PWA），群管理员分享链接
- 首字母登录码系统（如 ZS、CGT），无需注册
- 管理员后台分配账号和初始积分

## 4. 启动与部署

### 本地开发

```bash
# 1. 安装依赖
npm install

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 填入 API Keys

# 3. 启动开发服务器 (Express + Vite)
npm run dev
# 后端: http://localhost:3000
# 前端: 通过 Vite 代理

# 4. 类型检查
npm run lint    # tsc --noEmit
```

### 生产构建

```bash
npm run build   # vite build + esbuild server.ts
npm start       # node dist/server.cjs
```

### Docker 部署

```bash
docker-compose up -d
# Nginx(:80) + MySQL(:3306) + App(:3000)
```

### PM2 部署

```bash
pm2 start ecosystem.config.cjs
pm2 save
```

### 数据库模式

项目支持两种存储模式，由 `.env` 中 `APP_STORAGE_MODE` 控制：

| 模式 | 配置 | 适用场景 |
|------|------|---------|
| JSON 文件 | 默认（无 DATABASE_URL） | 开发/单机部署 |
| MySQL | `APP_STORAGE_MODE=mysql` + `DATABASE_URL` | 生产环境 |

## 8. 核心业务逻辑

### 竞猜流程

```
用户选择比赛 → 选择玩法(market) → 选择选项(optionKey) → 输入积分(stake)
→ 前端 POST /api/predictions  {matchId, market, optionKey, stakePoints}
→ prediction_service.ts: resolveOddsSnapshot() 获取赔率
→ 校验: 比赛状态、积分余额、单场限额
→ 写入 Prediction + Transaction (PREDICTION_STAKE)
→ WebSocket 广播竞猜事件
```

### 结算流程

```
比赛结束后:
→ settlement_service.ts: settleMatchById()
→ 依据 match.homeScore/awayScore 判定结果
→ 匹配 prediction.optionKey 判定输赢
→ 更新 prediction.status + settledProfit
→ adjustWalletBalance (PREDICTION_WIN/PREDICTION_LOSE)
→ WebSocket 推送结算结果 + 刷新排行榜
→ 检查徽章/称号/连胜
```

### 认证机制

- 用户使用**身份码**（如 WC1001）登录
- 可选 PIN 码二次验证
- 前端 localStorage 存储 loginCode
- 后端通过 `Authorization` header 读取 loginCode 匹配用户
- 管理员使用 `x-admin-token` header（会话管理）

### WebSocket 事件

| 事件 | 推送内容 |
|------|---------|
| `score_update` | 比分变化 |
| `prediction_result` | 竞猜结果 |
| `match_settled` | 比赛结算 |
| `notification` | 徽章/连胜通知 |
| `odds_change` | 赔率变动 |

## 9. 数据同步机制

### 赛程同步

- **来源**: API Football (`API_FOOTBALL_KEY`) 或静态种子数据
- **触发**: 定时任务 (cron) 或管理后台手动触发
- **端点和路由**: `POST /api/admin/sync/fixtures`

### 赔率同步（v2.3.1 新增手动模式）

- **来源**: The Odds API (`THE_ODDS_API_KEY`)
- **模式**: 仅手动触发（不自动同步以节省 API 调用次数）
- **端点**: `POST /api/admin/sync/odds`
- **请求**: `GET /v4/sports/soccer_fifa_world_cup/odds/?regions=eu&markets=h2h,totals`
- **限制**: 免费版每月 500 次调用
- **兜底**: 无 API Key 时使用 `generateDefaultOdds()` 基于 FIFA 排名生成

### 比分同步

- 通过赛程同步间接获取比赛状态和比分
- 降级模式：无 API Key 时需管理员手动录入比分

### 同步健康检查

`GET /api/health/sync` 返回各组件的健康状态和最近同步时间。

| 管理 | `AdminPanel` | 导航抽屉 → 用户/积分/备份/同步 |

### 性能优化

- Vite `manualChunks` 分离 5 个 vendor chunk (react/animation/charts/socket/canvas)
- React.lazy + Suspense 懒加载 7 个非首页组件
- 排行榜 O(n×(P+T)) → O(P+T+U) 单次遍历预聚合
- 图标使用 `lucide-react` 按需导入

## 12. 维护操作指南

### 手动同步赔率

```bash
# 1. 确保 .env 中配置了 THE_ODDS_API_KEY
# 2. 管理员登录后台
# 3. 在管理后台触发赔率同步（或调用 API）
curl -X POST http://localhost:3000/api/admin/sync/odds \
  -H "x-admin-token: <你的管理员token>"
```

### 数据备份

```bash
# 备份当前数据
curl -X POST http://localhost:3000/api/admin/backups \
  -H "x-admin-token: <token>" \
  -d '{}'

# 从备份恢复
curl -X POST http://localhost:3000/api/admin/backups/restore \
  -H "x-admin-token: <token>" \
  -d '{"fileName": "backup-2026-06-13.json"}'
```

### 手动结算比赛

```bash
# 设置比分并结算
curl -X POST http://localhost:3000/api/admin/matches/m-5/settle \
  -H "x-admin-token: <token>" \
  -d '{"homeScore": 2, "awayScore": 1}'
```

### 生产诊断

```bash
npm run ops:diagnose   # 运行诊断脚本
npm run ops:check:home-ai  # 检查首页 AI 状态
```

### 日志位置

- 应用日志: `logs/` 目录
- 备份文件: `backups/` 目录
- 同步日志: `/api/admin/sync-logs`
