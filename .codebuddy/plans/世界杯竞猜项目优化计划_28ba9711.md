## 用户需求

1. **比赛赛程同步**：检查已结束比赛（6月12日、13日）的比分和状态同步是否正常，无API key时降级逻辑有风险（0:0错误结算）
2. **焦点战逻辑互通**：确认焦点战倒计时、焦点战卡片、赛程栏焦点比赛逻辑一致，目前显示卡塔尔vs瑞士
3. **赔率同步缓存优化**：赔率目前大部分是兜底模板，需优化为优先使用同步数据缓存，保证赔率合理性（correctScore与h2h联动、统一两套赔率生成、触发WebSocket推送）
4. **项目功能稳定性审查**：修复同步失败无告警、降级比分逻辑风险、赔率推送未触发、前端数据不实时刷新等问题

## 产品概述

2026世界杯娱乐竞猜平台，基于 Express + Vite + React + TypeScript 构建，支持赛程同步、实时比分、赔率竞猜、自动结算等功能

## 核心功能

- 修复比赛同步降级逻辑，阻止无比分比赛错误结算
- 统一并优化赔率生成算法，correctScore赔率与h2h联动
- 接通WebSocket赔率变动推送，实现前端实时刷新
- 增加同步健康检查与异常告警机制
- 优化焦点战与赛程展示的数据一致性

## 技术栈

- 后端：Express.js + TypeScript（Node.js）
- 前端：React + TypeScript + Vite + Tailwind CSS
- 数据层：JSON 文件 / MySQL 双存储
- 实时通信：WebSocket（ws 库）
- 外部API：API-Football（赛程/比分）、The Odds API（赔率）
- 定时调度：node-cron + 自定义动态同步调度器

## 实施方案

### 核心问题与修复策略

**问题1：降级比分逻辑导致0:0错误结算（严重）**

- `sync_scheduler_service.ts` 降级逻辑中，比赛超过2小时自动标记FT，并通过 `broadcastScoreUpdate` 推送 `match.homeScore ?? 0`
- `settlement_service.ts` 中 `homeScore !== undefined` 检查可被 `?? 0` 绕过
- **修复**：降级标记FT时不设置比分（保持undefined），结算服务增加"无比分不结算"硬保护，降级推送比分时不传0:0

**问题2：两套 generateDefaultOdds 不一致**

- `utils/odds.ts` 固定 hw=2.20/d=3.20/aw=3.10，不考虑排名
- `sync.ts` 基于FIFA排名差异化，标记 `syncStatus: 'MANUAL_FALLBACK'`
- **修复**：统一到 `sync.ts` 版本（带排名参数），`utils/odds.ts` 改为调用统一函数

**问题3：correctScore 赔率与 h2h 不联动（关键优化）**

- h2h 已基于排名调整，但 correctScore 全是固定模板（所有比赛相同）
- **修复**：根据 h2h 隐含概率缩放 correctScore 模板赔率 — 强队主胜比分赔率降低，弱队客胜比分赔率升高
- 缩放算法：`adjustedOdds = baseOdds * (baseImpliedProb / actualImpliedProb)`，按 HOME_WIN/DRAW/AWAY_WIN 分组缩放

**问题4：WebSocket 赔率推送从未触发**

- `broadcastOddsChange`（websocket.ts:190）已定义但 `syncOddsForMatches` 中从未调用
- **修复**：赔率同步成功后，对比前后赔率变化，变化超过阈值时调用 `broadcastOddsChange`

**问题5：前端数据不实时刷新**

- HomeTab 仅 `useEffect` 初始化加载，不监听 WebSocket
- **修复**：App 层统一监听 WebSocket 事件，通过回调刷新各 Tab 数据

**问题6：同步失败无告警**

- API 调用失败仅记日志，管理员不知情
- **修复**：连续失败3次推系统通知；增加 `/api/health/sync` 健康检查端点

**问题7：服务启动不执行初始同步**

- 同步依赖 cron 每分钟 tick，首次执行有延迟
- **修复**：服务启动后立即执行一次完整同步 tick

**问题8：赔率来源不透明**

- 用户无法区分实时赔率和兜底模板
- **修复**：API 返回 `syncStatus` 和 `lastSyncedAt`，前端显示"实时赔率"或"参考赔率"标签

### 焦点战逻辑结论（无需修改）

审查确认：焦点战卡片、开赛倒计时、今日赛程均使用同一套 `selectFeaturedHomeMatch()` 算法，逻辑是互通一致的。当前显示卡塔尔vs瑞士是正确的（6月14日03:00是最近未开赛的比赛）。

### 实现要点

- `scaleCorrectScoreOdds` 新函数放在 `utils/odds.ts`，被 `sync.ts` 和初始数据生成共同调用
- 赔率WebSocket推送需对比新旧值，仅在变化超过0.05时推送，避免频繁推送
- 降级逻辑修复要兼容已有数据：对已以0:0结算的比赛不做回退，只防止未来再发生
- 健康检查端点返回：上次同步时间、同步状态、API配置情况、连续失败次数
