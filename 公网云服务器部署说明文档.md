# 公网云服务器部署说明文档

## 1. 部署目标

生产方案固定为：

- Ubuntu 22.04
- Docker + Docker Compose
- 同机 MySQL 8
- Nginx 反向代理
- 域名 + Let's Encrypt HTTPS
- 单房间模式
- 管理后台公网开放，但必须使用强密码

## 2. 服务器基础准备

```bash
sudo apt update && sudo apt upgrade -y
sudo timedatectl set-timezone Asia/Shanghai
sudo apt install -y ca-certificates curl gnupg lsb-release
```

## 3. 安装 Docker / Docker Compose

```bash
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo \"$VERSION_CODENAME\") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable docker
sudo systemctl start docker
```

## 4. 代码目录准备

```bash
sudo mkdir -p /srv/worldcup-app
sudo chown -R $USER:$USER /srv/worldcup-app
cd /srv/worldcup-app
```

将项目代码上传到该目录。

## 5. 域名解析

在域名服务商控制台添加 A 记录：

- `@` -> 云服务器公网 IP
- `www` -> 云服务器公网 IP（可选）

先确认：

```bash
ping example.com
```

## 6. 生产环境文件

复制示例文件：

```bash
cp .env.production.example .env.production
mkdir -p runtime deploy/certbot/www deploy/certbot/conf
```

重点修改：

```env
APP_DOMAIN=example.com
APP_URL=https://example.com

MYSQL_ROOT_PASSWORD=高强度Root密码
MYSQL_PASSWORD=高强度业务库密码
DATABASE_URL=mysql://worldcup_user:高强度业务库密码@mysql:3306/worldcup_app?charset=utf8mb4

ADMIN_USERNAME=admin_prod
ADMIN_PASSWORD=超长随机强密码
```

## 7. 放置旧数据

把原项目 `db.json` 放到：

```text
./runtime/db.json
```

## 8. 首次启动

```bash
docker compose --env-file .env.production up -d --build
```

查看状态：

```bash
docker compose --env-file .env.production ps
docker compose --env-file .env.production logs -f app
docker compose --env-file .env.production logs -f mysql
docker compose --env-file .env.production logs -f nginx
```

## 9. 初始化数据库结构

首次建表：

```bash
docker compose --env-file .env.production exec app npx prisma db push
```

后续升级版本：

```bash
docker compose --env-file .env.production exec app npm run prisma:migrate
```

## 10. 导入旧 JSON 数据

```bash
docker compose --env-file .env.production exec app npm run db:import:mysql
docker compose --env-file .env.production exec app npm run db:verify:mysql
```

## 11. Nginx 反代

项目已提供默认 HTTP 配置：

- [deploy/nginx/default.conf.template](/H:/世界杯娱乐项目/back/klpk-cgt-s-Org%20v2/deploy/nginx/default.conf.template)

首次签发证书前，Nginx 使用该 HTTP 配置即可完成：

- 80 端口访问
- 反代到 `app:3000`
- Let's Encrypt 验证目录暴露

## 12. Let's Encrypt HTTPS

先用 HTTP 配置启动成功后，执行证书签发：

```bash
docker run --rm \
  -v /srv/worldcup-app/deploy/certbot/www:/var/www/certbot \
  -v /srv/worldcup-app/deploy/certbot/conf:/etc/letsencrypt \
  certbot/certbot certonly --webroot \
  -w /var/www/certbot \
  -d example.com \
  -m you@example.com \
  --agree-tos \
  --no-eff-email
```

签发成功后切换 SSL 模板：

```bash
cp deploy/nginx/default-ssl.conf.example deploy/nginx/default.conf.template
docker compose --env-file .env.production restart nginx
```

## 13. 开放端口

云服务器安全组至少开放：

- `80/tcp`
- `443/tcp`

如果不需要直接暴露 MySQL，不要开放 `3306`。

## 14. 部署验收

检查以下项目：

- `https://example.com` 可正常打开
- 前端页面加载正常
- `/api/health` 返回 `ok=true`
- `db=mysql`
- 管理后台可登录
- MySQL 容器重启后数据仍在

## 15. 日志与重启

```bash
docker compose --env-file .env.production logs -f app
docker compose --env-file .env.production restart app
docker compose --env-file .env.production restart nginx
docker compose --env-file .env.production restart mysql
```

## 16. 备份建议

建议至少保留三类备份：

1. `runtime/db.json` 迁移源文件备份
2. MySQL 定时导出备份
3. `runtime/backups/` 应用侧 JSON 备份

MySQL 导出示例：

```bash
docker compose --env-file .env.production exec mysql \
  mysqldump -u"$MYSQL_USER" -p"$MYSQL_PASSWORD" "$MYSQL_DATABASE" \
  > mysql-backup-$(date +%F-%H%M%S).sql
```

## 17. 续期建议

Let's Encrypt 证书建议通过系统 `crontab` 定期续期，例如每月一次：

```bash
0 3 1 * * docker run --rm -v /srv/worldcup-app/deploy/certbot/www:/var/www/certbot -v /srv/worldcup-app/deploy/certbot/conf:/etc/letsencrypt certbot/certbot renew --webroot -w /var/www/certbot && cd /srv/worldcup-app && docker compose --env-file .env.production restart nginx
```
