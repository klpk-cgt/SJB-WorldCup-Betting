## 管理后台优化实施计划

### 需求分析与建议方案

#### 1. 删除账号和创建账号功能

**现状**：已有批量创建账号功能（`/api/admin/users/bulk`），但缺少单个创建和删除功能。

**方案**：
- 前端添加"单个创建账号"表单（输入昵称即可，自动生成 loginCode）
- 前端用户列表添加"删除"按钮（带二次确认）
- 后端新增 `DELETE /api/admin/users/:id` 接口（同时删除关联的 wallet、predictions、transactions）

#### 2. 为账号上传头像

**现状**：用户头像目前使用 emoji 字符串（`avatarUrl: ['⚽', '🏆', '🥅', ...]`），没有真实图片上传功能。

**方案**：
- 前端添加头像上传组件（支持拖拽/点击上传，图片预览，裁剪）
- 后端新增 `POST /api/admin/users/:id/avatar` 接口
- 图片存储方案建议：
  - **方案 A（推荐）**：将图片转为 Base64 存储在 `db.json` 的 `avatarUrl` 字段中（适合小规模群友使用，简单无需额外服务）
  - **方案 B**：上传到本地 `uploads/avatars/` 目录，`avatarUrl` 存储相对路径（迁移云服务器后需要配置静态文件服务）
  - **方案 C**：上传到第三方图床（如阿里云 OSS、腾讯云 COS），需要额外配置密钥
  
  考虑到你的项目只在群里朋友间流通，人数不多，**推荐方案 A（Base64）**，最简单且迁移 MySQL 后也能直接用。

#### 3. 后台中文提示报错日志

**现状**：部分错误提示是英文或不够友好，日志展示不够清晰。

**方案**：
- 统一后端所有 admin 接口的错误提示为中文
- 前端 Toast 提示优化，区分成功/警告/错误三种状态
- 日志页面增加筛选功能（按来源、状态筛选）
- 关键操作记录操作日志（谁在什么时间做了什么操作）

#### 4. 手动结算和调积分兜底

**现状**：已有调积分功能（`adjust-points`）和结算功能（`settle`），但可以增强。

**方案**：
- 调积分功能增加快捷模板（+500、+1000、+5000、-500 等快捷按钮）
- 增加"批量调积分"功能（一次给多人加积分）
- 结算增加"撤销结算"功能（结算错误时可以回滚）
- 增加积分流水查看功能（可以看到每个用户的积分变动历史）

#### 5. 首字母登录方案（核心需求）

**现状**：用户需要使用 `loginCode`（如 `WC1234`）登录，还需要 PIN 码。

**方案**：
- 创建账号时，自动生成昵称首字母拼音作为 `loginCode`（如"张三" → `ZS`，"老王" → `LW`）
- 新创建的账号不设 PIN（`pinHash` 为空），用户直接用首字母登录，无需 PIN
- 前端登录页增加"首字母快捷登录"模式（输入首字母即可进入）
- 后台创建账号时，自动用首字母作为 loginCode，管理员后续可以上传真实头像和修改昵称

**具体实现**：
```
管理员输入：张三
系统自动生成：
  - displayName: "张三"
  - loginCode: "ZS"（首字母大写）
  - pinHash: ""（空，不需要PIN）
  - status: "UNCLAIMED"
  
用户登录时：
  - 输入 "ZS" → 直接登录，无需PIN
```
